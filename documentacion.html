Descripción General del Proyecto

¿Qué es TD Nutrition y cuál fue el objetivo del proyecto?

🎯 Objetivo del proyecto

TD Nutrition es una tienda online completa de suplementos deportivos y equipamiento fitness, desarrollada como trabajo final de la materia Front-End. El objetivo fue demostrar dominio de las tecnologías modernas de desarrollo web: React 19, Vite, CSS Modules, React Router, Context API, Hooks personalizados, LocalStorage y diseño responsive premium.

🏋️
Categoría del producto

Tienda e-commerce especializada en suplementación deportiva (proteínas, creatina, pre-entreno, vitaminas, fat burners, mass gainers) y equipamiento de gimnasio (mancuernas, barras olímpicas, bolsas de boxeo, poleas y accesorios).

👨‍💻
Tipo de proyecto

Single Page Application (SPA) con navegación del lado del cliente, gestión de estado global sin librerías externas, autenticación simulada con persistencia en LocalStorage y diseño dark premium responsive.

Stack tecnológico completo del proyecto y justificación de cada elección.

Gestión del Estado — Context API

Cómo se maneja el estado global sin usar Redux ni Zustand.

🛒
CartContext

Estado: Lista de productos en el carrito, cantidades y total.

Funciones: addToCart(), removeFromCart(), updateQuantity(), clearCart(), totalItems, totalPrice.

Persistencia: Se guarda automáticamente en localStorage y se recupera al recargar la página.

🔐
AuthContext

Estado: Usuario logueado, rol (admin/user), estado de carga.

Funciones: login(email, password), register(name, email, pass), logout(), isAuthenticated, isAdmin.

Persistencia: Sesión guardada en localStorage. Al recargar mantiene la sesión activa.

🌙
ThemeContext

Estado: Tema actual (dark / light).

Función: toggleTheme() — alterna entre modo oscuro y claro aplicando el atributo data-theme al <html>.

Persistencia: Preferencia guardada en localStorage.

🔔
ToastContext

Estado: Cola de notificaciones activas.

Función: addToast(mensaje, tipo) — muestra notificaciones flotantes tipo "Producto agregado al carrito" o "Sesión cerrada". Se auto-eliminan a los 3 segundos.

App.jsx — Composición de Providers (envuelven toda la app)
export default function App() {
  return (
    <ThemeProvider>
      <AuthProvider>
        <CartProvider>
          <ToastProvider>
            <BrowserRouter>
              <Navbar />        // Accede a Cart + Auth + Theme
              <Routes> ... </Routes>
              <Footer />
            </BrowserRouter>
          </ToastProvider>
        </CartProvider>
      </AuthProvider>
    </ThemeProvider>
  );
}

Custom Hooks (Hooks Personalizados)

Lógica reutilizable extraída en hooks propios para mantener los componentes limpios.

📦
useProducts — Carga y filtrado de productos

Es el hook más importante del proyecto. Gestiona la carga asíncrona del catálogo, el filtrado por categoría y la búsqueda en tiempo real usando useMemo para evitar recálculos innecesarios.

hooks/useProducts.js — Lógica principal de productos
export function useProducts() {
  const [products, setProducts]     = useState([]);
  const [searchTerm, setSearchTerm] = useState('');
  const [activeCategory, setActive] = useState('all');
  const [loading, setLoading]       = useState(true);

  // Carga inicial — Fetch API simulada con delay
  useEffect(() => {
    fetchProducts().then(data => setProducts(data.products));
  }, []);

  // Filtrado en tiempo real (sin llamadas extra al servidor)
  const filteredProducts = useMemo(() => {
    let result = products;
    if (activeCategory !== 'all')
      result = result.filter(p => p.category === activeCategory);
    if (searchTerm.trim())
      result = result.filter(p => p.title.includes(searchTerm));
    return result;
  }, [products, activeCategory, searchTerm]);

  return { products, filteredProducts, loading, searchTerm,
             setSearchTerm, activeCategory, setActiveCategory };
}
HookHooks internos que usa¿Qué hace?
useProducts useState, useEffect, useCallback, useMemo Carga el catálogo, filtra por categoría y búsqueda en tiempo real.
useIntersectionObserver useRef, useState, useEffect Detecta cuando un elemento entra al viewport para disparar animaciones de aparición (fade-in al hacer scroll).
useScrollToTop useEffect, useLocation Hace scroll al tope de la página automáticamente cada vez que el usuario cambia de ruta.

Sistema de Autenticación

Login, registro y gestión de sesiones con Context API + LocalStorage.

💡 ¿Cómo funciona?

Como es un proyecto front-end sin backend real, la autenticación se simula con usuarios predefinidos en el código. En producción real, el login() haría un fetch() a un servidor. La estructura es idéntica: recibe email/contraseña, valida, guarda el usuario en el estado y en localStorage.

Flujo de autenticación:

  • Login: El usuario ingresa email + contraseña → se valida → se guarda en localStorage con la clave td_nutrition_auth → el Navbar muestra el avatar con las iniciales del usuario.
  • Persistencia: Al recargar la página, el AuthContext lee localStorage y restaura la sesión automáticamente usando useEffect.

Capa de Datos — Fetch API

Cómo se implementó la carga de datos con Fetch API simulando un servicio real.

🔄 Flujo de datos

Los datos de productos están en src/data/products.js como un array local. El archivo src/services/api.js los expone a través de funciones async que simulan la latencia de un servidor real (300-400ms de delay). Esto demuestra el uso correcto de Fetch API con async/await, manejo de errores y estados de carga.

services/api.js — Servicio de datos con delay simulado
// Simula latencia de red (como si fuera un servidor real)
const delay = (ms = 300) => new Promise(r => setTimeout(r, ms));

export const fetchProducts = async (limit = 100, skip = 0) => {
  await delay(400);            // Simula tiempo de respuesta
  const all = getAllProducts(); // Lee del catálogo local
  return { products: all.slice(skip, skip + limit), total: all.length };
};

export const searchProducts = async (query) => {
  await delay(200);
  const results = localSearch(query);
  return { products: results, total: results.length };
};

El hook useProducts consume este servicio usando Promise.all para cargar productos y categorías en paralelo, reduciendo el tiempo de espera total:

hooks/useProducts.js — Carga paralela con Promise.all
const [productsData, categoriesData] = await Promise.all([
  fetchProducts(100),  // Se ejecutan al mismo tiempo
  fetchCategories(),   // No se espera uno por uno
]);
setProducts(productsData.products);
setCategories(categoriesData);

Rutas y Navegación — React Router DOM

Configuración del enrutado del lado del cliente (SPA).

App.jsx — Definición de rutas con React Router v7
<Routes>
  <Route path="/"         element={<Home />}     />
  <Route path="/products" element={<Products />}  />
  <Route path="/about"    element={<About />}     />
  <Route path="/contact"  element={<Contact />}   />
  <Route path="/login"    element={<Login />}     />
  <Route path="*"         element={<NotFound />}  />  // 404
</Routes>
  • NavLink activo: El componente Navbar usa NavLink (no Link) para que el link de la página actual tenga un estilo diferente (naranja + underline).
  • Scroll al cambiar de ruta: El hook useScrollToTop escucha useLocation() y hace scroll al top automáticamente.
  • Ruta 404: El path "*" captura cualquier URL no definida y muestra la página de error personalizada.
  • vercel.json: Para que React Router funcione en Vercel, se configuró un rewrite que redirige todas las rutas a index.html.
